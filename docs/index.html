Flang Intermediate Language

A simple functional programming language

"F" is for flow.

"F" is for functional.

"lang" is for programming language.

Tutorial

Flang is a functional language, which means it's made of functions. Functions take input(s) and give output(s). Some people call these arguments and return values. Flang calls inputs and outputs "pipes." The outputs of functions are sent to inputs of others along pipes.

A function will wait until all of its inputs are "satisfied," which means they have been outputted by another function. Once its inputs are satisfied, it will run somepoint in the future. Once it runs, its outputs will become satisfied and the cycle will repeat.

Syntax

pipe ;function pipe pipe
pipe pipe pipe function pipe pipe
pipe function pipe pipe pipe
To be more specific: 
output ;lambda input input
input input input function output output
input function output output output

#include<stdio.h>

Precompiler Code

Flang compiles to C, which compiles to assembly, which then compiles to machine language. Compiling is translating a programming/computer language to another.

#include
tells our program to put the C code from /usr/include/stdio.h at the top of our program right before it is compiled from C to assembly. This is actually just C precompiler code, so it's standard and there are lots of resources on how to use it. In flang, anything that begins with a "#" (pronounced "pound") is precompiler code. The only other preprocessor function that is useful is "#define" which changes the last word on the line to the second.

All occourences of "Putchar" become "putchar" in the code above.

Lambdas

A lambda is a fancy word for a function that makes a function. In flang, the syntax that defines a function looks very similar to how you run a function, but the inputs are swapped with the outputs.

;main
means we are defining a function named "main"

All subsequent functions before the next lambda are the code in the function we are defining.
